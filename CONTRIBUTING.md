# Contributing

Thanks for contributing to the Portfolio Arcade project.

## Good contribution areas

- accessibility improvements
- bug fixes for layout, navigation, or game pages
- performance improvements for static assets
- cleaner code organization without changing the visual direction
- documentation improvements

## Before you open a pull request

1. Keep the portfolio identity consistent and professional.
2. Avoid adding unnecessary libraries for static-site problems.
3. Test the affected page on desktop and mobile.
4. Keep game additions lightweight and easy to understand.

## Pull request checklist

- explain the change clearly
- mention the page or game affected
- include screenshots when the UI changes
- keep the diff focused
