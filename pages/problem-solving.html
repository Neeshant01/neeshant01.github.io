Step-by-Step Approach

1. Define the actual issue

I first write down what is wrong in one sentence. This stops me from solving the wrong problem.

2. Change one variable

I test one meaningful change at a time so I can understand which action created the improvement.

3. Keep the best version

Once the solution works, I keep the cleaner and simpler version instead of adding unnecessary complexity.

Tips and Resources

Write the bug in plain language

If you cannot explain the issue simply, you probably need to understand it better before trying to fix it.

Use small experiments

One focused test gives more information than five random changes. This applies to coding and study both.

Keep notes on what failed

Failed attempts are useful because they remove wrong paths and make the next decision sharper.
